在Debug模式下可以进行调试。我们可以按F9设置断点。在设置妥当后，我们点击Windows调试器以确保我们是以调试的方式来执行此程序的。
## 黄色箭头和红色断点
开始调试之后，程序会停在第一个断点处，此时有一个黄色箭头在红色断点之上。这说明着即将执行这一句话（当前还没有执行）。
##  四种操作
在VS中有四个按钮对应四种操作让调试继续。

| 名称  | 效果            | 备注           |
| --- | ------------- | ------------ |
| 继续  | 到下一个断点        | 旁边有一个绿色的开始图标 |
| 逐语句 | 继续程序，会进入调用内部  | NULL         |
| 逐过程 | 继续程序，不会进入调用内部 | NULL         |
| 跳出  | 从当前跳出，返回到调用处  | NULL         |
## 内存视图
如果我们在内存视图中看到一个变量中全是c，那么说明他被创建了但是没有初始化。这将是一个重要的提示。
### 变动的断点
```CPP
int main() {
	int a; //⭕A
	return 0; //⭕B
}
```
我将断点设置在A处，但是当我开始调试的时候，断点自动变为了B。这是因为A处我声明的变量没有被使用，所以被编译器优化了。这和我们在空行上加断点是一样的道理。
