## 啥时候构造
|位置|代码|是否调用构造|说明|
|---|---|---|---|
|类成员|`Empty m_empty;`|❌ 不调用|只是声明|
|函数参数|`void func(Empty e)`|❌ 不调用|声明参数|
|函数内|`Empty alpha;`|✅ 调用无参|定义对象|
|函数内|`Empty beta(10);`|✅ 调用int构造|定义对象|
|函数内|`Empty gamma();`|❌ 函数声明|这是函数！|
|new|`Empty* p = new Empty;`|✅ 调用无参|动态创建|
|new|`Empty* p = new Empty(10);`|✅ 调用int构造|动态创建|
这是一个表格，里面有着各种情况下是否调用了构造函数。
## 例子
```CPP
class Entity
{
private:
	int m_x, m_y;
	Empty m_empty; // TAG1
public:
	Entity()
		: m_x(0), m_y(0), m_empty(3) // TAG2
	{
		std::cout << "Entity with default" << "\n";
	}
	Entity(int x, int y)
		: m_x(x), m_y(y), m_empty(3)
	{
		std::cout << "Entity with xy" << "\n";
	}
	void Print() const
	{
		std::cout << "x:" << m_x << " y:" << m_y << "\n";

	}
};
```
在TAG1处只是一个声明，并没有调用无参构造函数。 而在TAG2中才是真正的调用了有参构造。而TAG2这一行（从冒号开始）就是成员构造列表，你可以理解为将等号改为了圆括号。请注意，在成员初始化列表中，最好按照声明的顺序来，不然可能会产生意料之外的结果。
## 三种构造
```CPP
Empty a{2};
Empty b(2);
Empty c = Empty(3);
```
我们会建议使用第一种花括号，因为他不会产生歧义且禁止窄化转换。如果`b()`很可能会以为是一个函数声明。而第三种浪费资源生成了一个临时对象，可能会因为**拷贝省略**被改为`Empty c(3);`。
