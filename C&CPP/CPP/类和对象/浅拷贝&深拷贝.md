## 浅拷贝
**浅拷贝**是指只复制对象的基本类型成员和指针的**地址值**，而不复制指针所指向的资源。
### CPP默认行为
如果你**没有自定义**拷贝构造函数和赋值运算符，编译器会**自动生成**一个默认版本：
- **默认拷贝构造**：执行**浅拷贝**（逐个成员复制）
- **默认赋值运算符**：也执行**浅拷贝**
#### 如何判断是赋值还是拷贝构造？
判断标准是看**对象是否已经存在**。如果已经存在，那么就是赋值。否则为拷贝构造。
#### 示例代码
```CPP
class Simple {
public:
    int x;
    int* ptr;
    // 编译器自动生成：
    // Simple(const Simple& other) : x(other.x), ptr(other.ptr) {}  // 浅拷贝
    // Simple& operator=(const Simple& other) { x=other.x; ptr=other.ptr; return *this; }
};
```
#### 对于各种成员类型的行为

| 成员类型                     | 默认拷贝方式             |
| ------------------------ | ------------------ |
| 基本类型（int, float, char 等） | 值拷贝（浅拷贝）           |
| 指针                       | 拷贝地址（浅拷贝）          |
| 数组                       | 逐个元素拷贝（对于内置类型是浅拷贝） |
| 对象成员                     | 调用该对象的拷贝构造函数       |
```cpp

struct Student {
    int id;           // 值拷贝
    char* name;       // 指针地址拷贝（浅拷贝）
    int scores[5];    // 数组元素逐个拷贝
};
```
### e.g.
```CPP
#include <iostream>
class MyClass {
public:
    int* data;

    MyClass(int value) {
        data = new int(value);
    }
    
    // 使用默认的拷贝构造函数（浅拷贝）
};
int main()
{
	MyClass obj1{10};
    MyClass obj2 = obj1;
    
	*obj1.data = 20;
    std::cout << *obj2.data;
    
    return 0;
}
```
如果我们没有显示定义拷贝构造函数，那么会默认使用浅拷贝构造函数。
### 浅拷贝带来的问题
#### 双重删除（Double Delete）
```CPP
class MyClass {
public:
    int* data;
    
    MyClass(int value) {
        data = new int(value);
    }
    
    ~MyClass() {
        delete data;  // 析构时释放内存
    }
};
int main() {
    MyClass obj1(10);
    MyClass obj2 = obj1;  // 浅拷贝
    
    // obj1 和 obj2 析构时，同一块内存被释放两次！
    return 0;  // 运行时错误：double free
}
```
#### 数据共享和意外修改
```CPP
MyClass obj1(10);
MyClass obj2 = obj1;
*obj1.data = 100;  // 修改 obj1
std::cout << *obj2.data;  // 输出 100，obj2 也被修改了
```

## 禁止一切构造
```CPP
MyClass(const MyClass&) = delete;
MyClass& operator=(const MyClass&) = delete;
```
这样就既不能用拷贝构造，又不能用拷贝赋值了。
