## 两种条件编译方式
要是我们想要某些调试代码在Debug模式下启用，我们可以在本项目的C/C++下的预处理器中添加自己的预处理器定义。
### `ifdef`的方法
例如在Game项目中，我们可以定义`GA_DEBUG`。然后在代码中判断：
```CPP
#include <iostream>
int main()
{
	#ifdef GA_DEBUG
		std::cout << "Debug mode is on." << std::endl;
	#endif
}
```
也就是说如果在Debug模式下，就会输出Debug模式开启的信息。**但是这是不推荐的的做法**，假如说一个人以为输出调试是由1/0开启的，那么这将无法关闭调试信息。
### `if`的方法
例如在Game项目中，我们可以给Debug模式下定义`GA_DEBUG=1`，且Release模式下定义`GA_DEBUG=0`接下来，只要执行这个代码就可以了：
```CPP
#include <iostream>
int main()
{
	#if GA_DEBUG == 1
		std::cout << "Debug mode is on." << std::endl;
	#endif
}
```
## 替换时的注意事项
如果一定要替换代码，为了风格统一，我们一般在宏处不加分号，而在具体替换处加分号。
