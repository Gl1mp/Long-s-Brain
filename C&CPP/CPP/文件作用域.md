**文件作用域 = 写在所有函数、类、命名空间外面的东西**。
```CPP
int a = 10;           // ✅ 文件作用域

int main() {
    int b = 20;       // ❌ 不是文件作用域（在函数内）
}

class MyClass {
    int c = 30;       // ❌ 不是文件作用域（在类内）
};

namespace NS {
    int d = 40;       // ❌ 不是文件作用域（在命名空间内）
}
```
对于这些定义语句例如`class MyClass`则是定义在文件作用域的，但类内部的成员属于类作用域。**而正是因为他定义在了文件作用域上 ，整个文件中各处都可以访问到他。**
## 特点

| 特征       | 说明                                      |
| -------- | --------------------------------------- |
| **位置**   | 必须在任何 `{}` 之外（函数体、类体、命名空间体都不行）          |
| **作用范围** | 从声明位置到文件结束，这个文件里的所有函数都能看到它              |
| **跨文件**  | 默认情况下，其他文件也能访问（除非加了 `static` 或放在匿名命名空间） |
## 其他作用域
```CPP
int file_var = 1;                 // 文件作用域（整个文件都能用）

namespace MyNS {
    int ns_var = 2;               // 命名空间作用域（必须加 MyNS::）
}

class MyClass {
    int member_var = 3;           // 类作用域（必须通过对象访问）
    static int static_member;     // 类作用域（通过 MyClass:: 访问）
};

void func() {
    int local_var = 4;            // 块作用域（只在 {} 内有效）
}

int main() {
    // 只有 file_var 是直接可用的文件作用域变量
}
```
## Summary
**文件作用域就是写在最外层、不套在任何 `{}` 里面的声明。**
- 在 C 语言里，这叫“文件作用域”
- 在 C++ 里，标准术语是“全局命名空间作用域”
