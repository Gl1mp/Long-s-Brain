编译系统是指预处理器、编译器、汇编器、链接器。当我们提及编译器的时候，我们通常是指整个编译器系统（或预处理器、编译器、汇编器）。**本文的所有适用于Windows中的MSVC**。
## 流程

| 阶段  | 工具                          | 输入文件           | 主要输出文件                              | 备注                         |
| --- | --------------------------- | -------------- | ----------------------------------- | -------------------------- |
| 预处理 | cl.exe (/P)                 | .cpp / .c / .h | **.i**                              | 展开宏、包含头文件、条件编译后纯文本         |
| 编译  | cl.exe                      | .i（或直接.cpp）    | **.obj**（默认不输出.asm） 可选 .asm（用 /FAs） | 把 C++ 转成机器码，生成 COFF 格式.obj |
| 汇编  | cl.exe 内置 或 ml.exe/ml64.exe | .asm（如果有）      | **.obj**                            | MSVC通常不单独输出.asm 再汇编        |
| 链接  | link.exe                    | .obj + .lib    | **.exe** / **.dll** / **.lib**      | 合并所有.obj，解析符号，生成可执行文件或库    |
```mermaid
flowchart LR
    G[开始]-->|.cpp/c./.h| A[预处理]
    A[预处理] -->|.i| B[编译]
    B -->|.asm*| C[汇编]
    C -->|.obj| D[链接]
    D -->|.exe/.dll/.lib| F[结束]
```
注：有星号的默认不输出。
## MSVC的特点
cl.exe（MSVC 的编译驱动）内部包含了：
- 前端（c1.dll / c1xx.dll）：解析 C/C++ 语法、语义分析、生成中间表示
- 后端（c2.dll）：优化 + 代码生成 → 直接产生机器码（COFF 格式的 .obj 文件内容）

也就是说编译和汇编都交给了cl，中间不再输出.asm文件。
