1. 从理论上来讲，他们都能够找到正确的头文件（如果设置位置正确的话）。
2. `<>`会去编译器系统路径找。`""`会先去当前文件所在目录找。如果未找到，那么会去编译器系统路径找。
3. 换一个思路，我们如果把当前文件目录加到编译器路径中，两者就是等价的了。所有并没有那么多区别，只要能够区别三方库、标准库和本项目的库即可。在
## 规则
在约定俗成下，我们应该遵循：
- **`<>`**：不是本项目维护的代码（标准库、第三方库）
- **`""`**：本项目内维护的头文件
## 示例
```CPP
// 标准库
#include <iostream>
#include <vector>
// 第三方库（如 Qt、OpenCV）
#include <QApplication>
#include <opencv2/core.hpp>
// 项目内部头文件
#include "utils/logger.h"
#include "config.h"
```
