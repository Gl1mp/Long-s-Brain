命名空间的作用或初衷就是为了防止函数名重复。我们可以在不同的命名空间下，使用相同函数签名的函数并通过命名空间将其区分开来。
## C中的命名空间替代
C中并没有引入命名空间，所以一些C库都会在自己的函数、结构体前加上独有的特征标识符。例如有个库叫`HLF`，那么很可能他的所有函数、结构体都有前缀`hlf_`。
## 使用典例
```CPP
#include <iostream>
namespace orange
{
	namespace blue_orange
	{
		//std::cout << "blue_orangeHere!"; POINTα
		void Show()
		{
			std::cout << "blue_orangeHere!\n";
		}
	}
}

namespace banana
{
	void Show()
	{
		std::cout << "bananaHere!\n";
	}
}

namespace strawberry::red_strawberry
{
	void Show()
	{
		std::cout << "RedStrawbeeryHere!\n";
	}
}
int main()
{
	{
		using namespace orange::blue_orange;
		Show();
	}
	
	{
		using namespace banana;
		Show();
	}
	
	{
		using namespace orange;
		blue_orange::Show();
	}
	
	{
		orange::blue_orange::Show();
	}
	
	{
		banana::Show();
	}
	
	{
		strawberry::red_strawberry::Show();
	}
}
```
这个例子中涵盖了你需要知道的基本示例。
- 请注意`Pointα`处的代码是非法的，我们不能直接在命名空间中书写这一类语句，而应该将其放在函数、结构体等等中
- 我们如果使用`using namespace orange`，只能找到`orange`直接包含的东西，而不能找到他的子命名空间中的东西
