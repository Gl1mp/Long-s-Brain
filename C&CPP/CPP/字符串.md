## 单双引号的区别
单引号是表示单个字符，而双引号则是代表着一个字符串，或者是以`\0`结尾的字符数组。对于`"Abc"`而言，他的类型为`const char[4]`，但是在传递使用的时候会发生隐式退化，退化为指向首元素的指针，类型为`const char*`。
## 两种风格
### C风格
```CPP
	const char* str1 = "HelloWorld!";
	std::cout << str1;
```
### CPP风格
```CPP
#include <iostream>
#include <string>
int main(){
	std::string str2 = "HelloWorld!";
	std::string str3 = str2 + "ByGl1mp!";
	std::cout << str3 << "\n";
	std::cout << str3.size() << "\n";
}
```
#### 头文件问题
`<iostream>` 中包含了 `std::string` 的前置声明，但 `std::string` 的完整定义和所有操作符重载（包括 `<<` 和 `>>`）是在 `<string>` 头文件中定义的。
#### 重载符
这里实际上发生了两种重载：
-  `operator+` 重载（字符串连接）
-  `operator<<` 重载（输出到流）
## 字符串字面量
"abcdef"就是个字符串字面量。他本身是个`const char[]`类型，可以赋值给一个`const char* ptr`。
### C风格
```CPP
const char* str1 = "Hello!";
std::cout << str1 << '\n';
```
这样可以输出整个字符串，请注意输出的时候传入的是指针而非解引用后的结果。
### CPP风格
```CPP
const std::string str2 = "Hello!";
std::cout << str2 << '\n';
```
## std::string_literals
`using namespace std::string_literals` 允许你使用 `s` 后缀来创建 `std::string` 对象，而不是C风格的字符串。
```CPP
using namespace std::string_literals;
const std::string str3 = "Hello"s + "World!";
std::cout << str3 << '\n';
```
