定义在`<memroy>`头文件中的智能指针有三类。
## std::unique_ptr

**独占**它所指向的对象。同一时间，只能有一个 `unique_ptr` 指向一块内存。它不能被拷贝，只能被移动。开销几乎为0，效率很高。如果可以的情况下我们尽可能使用这个。
### 定义方法
```CPP
std::unique_ptr<Entity> entity = std::make_unique<Entity>();
```
我们应该尽量使用这种构造方法而不是`std::unique_ptr<Entity> entity(new Entity());`如果在`unique_ptr`构造函数之前如果发生了错误就会导致内存泄露。
## std::shard_ptr
**共享**它所指向的对象。它使用**引用计数**来记录有多少个 `shared_ptr` 指向同一个对象。每当一个新的 `shared_ptr` 指向该对象，引用计数加1；每当一个 `shared_ptr` 被销毁，引用计数减1。当最后一个 `shared_ptr` 被销毁（引用计数变为0）时，对象被自动删除。相较于`unique_ptr`，他需要额外的一块内存空间来存放引用计数。
### 定义方法
```CPP
std::shared_ptr<Entity> sharedEntity= std::make_shared<Entity>();
```
我们仍然不推荐使用形如`std::shared_ptr<Entity> sharedEntity(new Entity());`的方式。与上文不同的是，如果这么做会带来更多的资源开销。因为`shard_ptr`需要控制块来存放其引用计数，如果先`new`了一块区域那么`shard_ptr`构造函数还需要再开辟一块区域。
```CPP
std::shared_ptr<Entity> e2 = sharedEntity;
```
我们可以通过此来自由复制。
## std::weak_ptr
`weak_ptr` 是一种**不控制对象生命周期**的智能指针。它指向一个由 `shared_ptr` 管理的对象，但**不会增加引用计数**。它就像一个"观察者"，可以查看对象是否还存在，但不能直接操作它（需要先提升为 `shared_ptr`）。
### 定义方法
```CPP
std::weak_ptr<Entity> e3_w = e2;
```
接[[#std shard_ptr]]中的例子，我们可以直接将其赋值给一个`weak_ptr`，我们在使用前可以询问一下这个指针是否依旧有效。
