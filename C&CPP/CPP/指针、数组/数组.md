当我们索引数组中的某个元素时，我们会得到一个有基础数据类型的变量。如果有`int array[4]`，那么我们索引`array[0]`时就得到了一个数据类型为`int`的变量。
## 类型之谜
仍以`int array[4]`为例，我们可以在VS的调试界面中看到array的类型是`int[4]`，但是我们在进行指针。
```CPP
int examples[10];
	std::cout << examples << "\n";
	std::cout << examples + 1 << "\n";
```
### 第一句的解释
`examples`在此处的确发生了退化，这是因为这个函数没有数组版本的`operator<<`，重载选择了`cout << int*`，所以将首元素地址打印了出来。
### 第二句的解释
此处，`examples`本身类型是`int[10]`。在指针加法的时候退化为了`int*`类型，加一后就指向了第二个元素。如果此时解引用就是`examples[1]`中的内容。
## 维护长度
数组的长度对其是相当重要的，我们上面所展示的都是C中的字符串形式。
### sizeof
正是因为他的类型是`int[4]`，所以当我们对其使用`sizeof`的时候，得到的是整个字符串的大小。如果我们除以基本类型的大小，那么就可以得到数组的长度了，也就是数组中元素的个数了。
### count&size
我们一般用`count`来描述数组中元素的个数，用`size`来描述数组的大小。当我们想要固定数组长度并以此长度定义数组时，一般都需要将这个`count`定为`static`变量。需要注意的是：**C++ STL 容器中`size()` 返回元素个数。**如果要更清楚的说明，我们可以用`byteSize`来命名。
### 传参退化
当我们进行传参的时候，`int[4]`这样的类型会退化为首地址的指针。也就是说`sizeof`不能得出整个数组的大小了。
### CPP中引入的新的数组
C++ STL 容器中`size()` 返回元素个数。
```CPP
#include <iostream>
#include <array>
int main(){
	std::array<int, 5> arr;
	std::cout << arr.size() << "\n";
}
```
