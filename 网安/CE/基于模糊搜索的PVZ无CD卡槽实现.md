模糊搜索是指在不知道精确值的情况下进行的搜索。
## 1.选择进程
选择进程`0054A78-PlantsVsZombies.exe`。
## 2.开始扫描
选择`未知初始值`，并进行首次扫描。
## 3.筛选
我们可以有一下几种方式筛选掉不可能的地址。
- 在不影响卡槽CD的情况下让游戏继续几秒，然后暂停游戏，扫描类型设置为`未改变的值`，点击`下一次扫描`。
- 放置植物，然后暂停游戏，让扫描类型设置为`变动的值`，点击`下一次扫描`。
重复这两种方式，最终可以筛选出少量地址。我们发现在游戏暂停时，除了一个地址，其他都在变化，明显其他的都不符合逻辑（游戏暂停的时候冷却时间应该不变的）。
## 4.冷却逻辑
我们现在查找到的是第一个卡槽的冷却时间，发现他从0开始不断增长到了一个确定的值后又变成0并保持不变。也就是说他的冷却完成了。请注意，不能将冷却时间直接修改为0，应该将其修改为一个很大的数，然后让游戏把其修改为0。
## 5.查找其他槽位的CD地址
第三步的筛选是麻烦且枯燥的。当我们知道第一个槽位地址后，就不需要再重复这个过程了。选中第一个卡槽的地址，右击选择`浏览此内存区域`。继续游戏，放置第二个卡槽的植物后观察此区域。
![[Picture/Pasted image 20260306163324.png]]
会发现红色的地方一直在增大（这是十六进制数，你也可以右键修改位十进制数）。右击他，选择`将此地址添加到列表`。卡槽的地址增量一般都是固定的，我们可以发现差值为50（十六进制）。那么我们也可以算出卡槽3的地址，依次类推。
## 6.编写自动化脚本
查看写入第一个槽地址的汇编指令，在拿起这个植物的时候出现第一行指令，在放下的时候出现第二行指令。我们需要的是第二行的指令（因为拿起来并不影响冷却时间）。
![[Picture/Pasted image 20260306165057.png]]
然后我们选中第二条，将其在反汇编器中显示。用Ctrl+A进入自动汇编，选择`模板`>`作弊表框架代码`，`模板`>`代码注入`。
```
originalcode:
add dword ptr [edi+24],01
mov eax,[edi+24]
```
我们发现add这一条是对指针对应内存区域加1。那么我们可以直接加一个很大的值，来快速超过阈值。
```
add dword ptr [edi+24],#5000
```
\#代表这是一个十进制值。然后选择`文件`>`分配到当前作弊表`，回到CE主界面，选择激活这个脚本，就可以实现无CD啦。
## 7.原理
第六步中，我们从第一个卡槽的汇编代码入手，写了一个脚本。我们使用时发现，所有卡槽都实现了无CD，这是为什么呢？
### TICK&循环
游戏每帧（tick）都会跑一个**循环**，遍历当前的所有卡槽（最多 10 个）。循环中有一条汇编指令就是在负责给冷却计数器＋1。
```
inc [edi+24]         //或者 add dword ptr [edi+24], 1
```
edi 指向当前正在处理的卡槽结构，+24 是冷却计时器的偏移。我们修改的这个指令是所有槽共用的核心循环指令。
