本文是[[4aA.一级指针下的手动查找基址（CE教程第六关）]]的升级版，可以先阅读此文章。在寻找到了首地址后，查看详细信息得到：
```
10004910A - 89 46 18  - mov [rsi+18],eax
RSI=0161E440
```
偏移量为`18(HEX)`，意思为将`EAX`中的值放到`RSI+18`计算出的地址所指向的内存空间中。
接下来请阅读[[CE的指针称呼]]。
## 追查
现在我们知道了首地址`0161E458`，对其解引用就是阳光的值。这个RSI中的值并非空穴来风，而是从其他地方读取过来的，所以我们要查找哪个地址对应的内存区中存储着这个指针。
```
1000490BC - 48 83 3E 00 - cmp qword ptr [rsi],00
1000490C8 - 48 8B 36  - mov rsi,[rsi]
RSI=062355E0
```
我们会发现在此处偏移为0，继续追查RSI。
```
100049070 - 48 83 7E 18 00 - cmp qword ptr [rsi+18],00
10004907C - 48 8B 76 18  - mov rsi,[rsi+18]
RSI=016846A0
```
同上，只是此处偏移变成了18。继续追查RSI。
```
100049028 - 48 83 7E 10 00 - cmp qword ptr [rsi+10],00
100049034 - 48 8B 76 10  - mov rsi,[rsi+10]
RSI=06235120
```
偏移变成了10。再追查下去就只有一条记录了，找到了一个绿色的基址。
## 创建指针链
双击基址的地址，勾选中`指针`。点击若干次`添加偏移量`，从下往上指针级数递增。我们依次将`10,18,0，18`从下至上填入（ps：这些都是HEX）。这样就完成了一个指针链。即使游戏重启，我们依旧可以精准找到目标变量的地址。
## 锁定数值
选择`查找写入此地址的内容`，再选择`查找写入此指针指向地址的代码`，可以直接将汇编代码NOP掉，最后修改数值直接锁定即可。
## 代码注入
通过`查找写入此地址的内容`可以抓到汇编指令，再右键进行反汇编。选中此汇编指令，`ctrl+A`进行自动汇编。选择模板中的代码注入，跳转到此处，就可以进行修改了。
