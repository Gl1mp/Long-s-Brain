这是教程的最后一关，当定位四个血量后，都有这么一句汇编指令
```
1000499ED - F3 0F11 43 08  - movss [rbx+08],xmm0
```
也就是说这句话被共享了。`xmm0`中的值就是`[rbx+08]`的值。
## 寻找
我们不用一个个锁定血量对应的地址了，只要在锁定一个后打开反汇编，右击这句汇编，选择`查找此指令访问的地址`，然后再分别打四下，四个血量的地址就全部显示了。将他们全部选中，可以添加到地址栏中。
## 数据结构
在`查找此指令访问的地址`界面中，将他们四个选中，右击`使用选中地址打开解析数据`。
![[Pasted image 20260307144625.png]]我们发现己方阵营编号为1，敌方编号为2。
## 汇编注入
接下来我们就直接注入到这条汇编指令，选择`自动汇编`，再选择`模板`中的`作弊代码框架`，`模板`中的`代码注入`。下面是代码示例：
```
[ENABLE]
alloc(newmem,2048,FFFF000F) 
label(returnhere)
label(originalcode)
label(exit)

newmem:
cmp byte ptr [rbx+14], 1       // 判断是否是队1
je team1                       // 如果是队1，跳到team1
cmp byte ptr [rbx+14], 2       // 判断是否是队2
je team2                       // 如果是队2，跳到team2
jmp originalcode               // 都不是，执行原指令

team1:
movss xmm0, [value1000]        // 队1 → 1000
jmp originalcode

team2:
xorps xmm0, xmm0               // 队2 → 0 (最快清零)
// 或者用：movss xmm0, [value0]
jmp originalcode

originalcode:
movss [rbx+08], xmm0           // 原指令

exit:
jmp returnhere

value1000:
dd (float)1000.0

// value0:
// dd (float)0.0                // 如果用movss方式清零，取消注释这行

FFFF000F:
jmp newmem
returnhere:

[DISABLE]
dealloc(newmem)
FFFF000F:
movss [rbx+08], xmm0           // 恢复原指令
db F3 0F 11 43 08              // 机器码备份
```
这个代码涉及到了汇编编程，不做过多要求。
