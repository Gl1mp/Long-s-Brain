在使用 Cheat Engine (CE) 修改器进行内存修改时，汇编指令（Assembly Instructions）是核心操作的基础。以下是一些在 CE 中常见的基础汇编指令。
### 1. MOV (Move) - 数据移动指令
**功能**：将数据从一个位置（寄存器、内存）移动到另一个位置。
**语法**：`MOV 目标, 源`
**例子**：
```assembly
MOV EAX, [EBX]       ; 将 EBX 指向的内存地址中的值移动到 EAX 寄存器
MOV [0x00401000], 100 ; 将立即数 100 写入内存地址 0x00401000
```
**讲解**：
- **用途**：最常用的指令，用于读取或修改内存值。例如，修改游戏中的生命值。
- **CE 场景**：假设你找到生命值的内存地址为 `0x00401000`，可以通过 `MOV [0x00401000], 999` 将生命值改为 999。
- **注意**：`[]` 表示内存地址，`EAX` 是通用寄存器。移动不会改变源数据。
### 2. ADD (Add) - 加法指令
**功能**：将两个值相加，结果存入目标位置。
**语法**：`ADD 目标, 源`
**例子**：
```assembly
ADD EAX, 10          ; EAX 的值增加 10
ADD [0x00401000], 50 ; 内存地址 0x00401000 的值增加 50
```
**讲解**：
- **用途**：用于增加寄存器或内存中的值，如增加游戏金币。
- **CE 场景**：如果金币地址是 `0x00401000`，可以用 `ADD [0x00401000], 1000` 增加 1000 金币。
- **注意**：可能会触发溢出，需注意数据类型（如 32 位整数）。
### 3. SUB (Subtract) - 减法指令
**功能**：从目标值中减去源值，结果存入目标位置。
**语法**：`SUB 目标, 源`
**例子**：
```assembly
SUB EAX, 5           ; EAX 的值减少 5
SUB [0x00401000], 20 ; 内存地址 0x00401000 的值减少 20
```
**讲解**：
- **用途**：用于减少数值，如减少游戏中的弹药量。
- **CE 场景**：若想减少敌人血量（地址 `0x00402000`），可以用 `SUB [0x00402000], 50`。
- **注意**：结果可能为负数，需确认游戏逻辑是否允许。
### 4. CMP (Compare) - 比较指令
**功能**：比较两个值，设置标志寄存器（Flags），通常用于条件跳转。
**语法**：`CMP 目标, 源`
**例子**：
```assembly
CMP EAX, 100         ; 比较 EAX 和 100
CMP [0x00401000], EBX ; 比较内存地址 0x00401000 的值和 EBX 的值
```
**讲解**：
- **用途**：不修改数据，仅影响标志位（如零标志 ZF、符号标志 SF），常与跳转指令搭配。
- **CE 场景**：用于分析游戏逻辑，比如检查生命值是否为 0（`CMP [0x00401000], 0`），然后跳转到死亡处理代码。
- **注意**：常与 `JMP`、`JE`、`JNE` 等指令配合。
### 5. JMP (Jump) - 无条件跳转指令
**功能**：无条件跳转到指定地址执行代码。
**语法**：`JMP 目标地址`
**例子**：
```assembly
JMP 0x00401500       ; 跳转到地址 0x00401500
```
**讲解**：
- **用途**：改变程序执行流程，如跳过某段代码（例如扣血逻辑）。
- **CE 场景**：若游戏在 `0x00401000` 处有扣血代码，可以用 `JMP 0x00401010` 跳过它。
- **注意**：需确保目标地址有效，否则可能导致程序崩溃。
### 6. JE/JZ (Jump if Equal/Zero) - 条件跳转（相等/零）
**功能**：如果比较结果相等（或值为零），则跳转。
**语法**：`JE 目标地址`
**例子**：
```assembly
CMP EAX, 0
JE 0x00401500        ; 如果 EAX 等于 0，跳转到 0x00401500
```
**讲解**：
- **用途**：常用于检查条件，如生命值是否为 0。
- **CE 场景**：若想让生命值耗尽时不触发死亡，可以将死亡跳转指令替换为 `NOP` 或其他逻辑。
- **注意**：依赖 `CMP` 设置的标志位。
### 7. JNE/JNZ (Jump if Not Equal/Not Zero) - 条件跳转（不等/非零）
**功能**：如果比较结果不相等（或值非零），则跳转。
**语法**：`JNE 目标地址`
**例子**：
```assembly
CMP [0x00401000], 0
JNE 0x00401500       ; 如果内存值不等于 0，跳转到 0x00401500
```
**讲解**：
- **用途**：用于跳过某些逻辑，如“生命值为 0 则游戏结束”。
- **CE 场景**：可用来绕过游戏失败条件。
- **注意**：与 `JE` 相反，需结合 `CMP` 使用。
### 8. NOP (No Operation) - 空操作指令
**功能**：什么都不做，占位指令。
**语法**：`NOP`
**例子**：
```assembly
NOP                  ; 空操作
```
**讲解**：
- **用途**：用于覆盖原有指令，禁用某段代码逻辑。
- **CE 场景**：若游戏扣血代码是 `SUB [0x00401000], 10`，可以用 `NOP` 替换，防止扣血。
- **注意**：NOP 占 1 字节，需确保覆盖长度匹配原指令。
### 9. PUSH - 入栈指令
**功能**：将数据压入堆栈。
**语法**：`PUSH 源`
**例子**：
```assembly
PUSH EAX             ; 将 EAX 的值压入堆栈
PUSH 0x100           ; 将立即数 0x100 压入堆栈
```
**讲解**：
- **用途**：保存寄存器值或传递函数参数。
- **CE 场景**：在注入自定义代码时，可能需要保存寄存器状态。
- **注意**：堆栈操作会影响 `ESP` 寄存器。
### 10. POP - 出栈指令
**功能**：从堆栈顶部弹出数据到目标。
**语法**：`POP 目标`
**例子**：
```assembly
POP EAX              ; 将堆栈顶部值弹出到 EAX
```
**讲解**：
- **用途**：恢复寄存器值或获取函数参数。
- **CE 场景**：与 `PUSH` 配合，用于临时保存和恢复数据。
- **注意**：确保堆栈平衡，否则可能导致崩溃。
### 11. CALL - 调用函数指令
**功能**：调用子程序，保存返回地址后跳转。
**语法**：`CALL 目标地址`
**例子**：
```assembly
CALL 0x00402000      ; 调用地址 0x00402000 处的函数
```
**讲解**：
- **用途**：执行游戏中的函数，如刷新界面。
- **CE 场景**：可调用游戏内置函数来实现特定效果。
- **注意**：需确保函数地址正确，且返回地址被正确处理。
### 12. RET - 返回指令
**功能**：从子程序返回，跳转到调用者的返回地址。
**语法**：`RET`
**例子**：
```assembly
RET                  ; 返回到调用者
```
**讲解**：
- **用途**：结束函数执行。
- **CE 场景**：在注入代码时，需确保函数正确返回。
- **注意**：堆栈状态需正确，否则返回地址可能错误。
### 13. INC（Increment）—— 加 1（自增）
**功能**：把目标操作数的值 **+1**，结果写回原位置。

**语法**：
```assembly
INC 寄存器
INC 内存地址
```

**常见写法示例**（CE 里最常看到的几种）：
```assembly
INC EAX             ; EAX ← EAX + 1
INC EBX
INC ECX             ; 计数器最常见
INC EDX

INC [ESI+0x14]      ; 把 ESI+0x14 这个内存地址的值 +1
INC DWORD PTR [EBX+000004A4]   ; CE 里经常出现的长写法
INC byte ptr [eax+30]          ; 只对 1 字节自增
```

**影响的标志位**（Flags）：
- ZF（零标志）、SF（符号标志）、OF（溢出标志）、AF（辅助进位）、PF（奇偶标志）**都会被更新**
- **CF（进位标志）不受影响** ← 这点和 ADD 不同，是 INC 的一个重要特性
### 14. DEC（Decrement）—— 减 1（自减）
**功能**：把目标操作数的值 **-1**，结果写回原位置。

**语法**：
```assembly
DEC 寄存器
DEC 内存地址
```

**常见写法示例**：
```assembly
DEC EAX             ; EAX ← EAX - 1
DEC ECX             ; 非常常见的倒计时/剩余次数
DEC [EDI+0x8]
DEC DWORD PTR [EBP-0x10]
DEC byte ptr [ebx+00001234]
```

**影响的标志位**：
- 和 INC 一样，**CF 不受影响**，其他常用标志位（ZF、SF、OF、AF、PF）都会更新

**CE 里最常见的用途**：
1. 生命值/护盾/耐久度 -1
2. 弹药/箭/金币/剩余次数 -1
3. 倒计时（timer--）
4. 循环计数器（倒着数）
5. 血量条递减、怒气值递减、冷却时间递减
---
### 典型 CE 修改示例
假设你想修改游戏生命值不减少：
1. **找到扣血代码**：通过 CE 扫描找到扣血逻辑，例如：
   ```assembly
   SUB [0x00401000], 10 ; 生命值减 10
   ```
2. **注入 NOP**：用 CE 的代码注入功能，将 `SUB` 替换为：
   ```assembly
   NOP
   NOP
   ```
   （`SUB` 可能占 2 字节，需用 2 个 `NOP` 填充）。
3. **验证效果**：运行游戏，确认生命值不再减少。
---
### 注意事项
- **指令长度**：每条指令占用字节数不同（`NOP` 1 字节，`MOV` 可能 2-7 字节），修改时需保持字节对齐。
- **寄存器保护**：注入代码时，需用 `PUSH` 和 `POP` 保存和恢复寄存器状态。
- **调试工具**：CE 的“内存视图”可查看和修改汇编代码。
- **备份**：修改前备份游戏文件，防止崩溃。
