SSH是一个用来远程加密登录的工具，但是实际上我们还可以用此创建SSH隧道。
## SSH会话
以本地机上有SSH客户端，服务器上有SSH服务端。当两者连接后，就建立了SSH会话。在通信时，机器上的防火墙只能知道这是SSH数据包，但不知道具体内容（已被加密）。也就是说，SSH让客户端和服务端之间的通信以加密方式进行。

### SSH本地隧道
#### 建立隧道
原始命令
```
ssh -i ~/.ssh/siyao.pem ubuntu@44.200.72.244
```
-i后面指定了私钥的位置。后面的格式为用户名@跳板机地址。接下来我们把本地隧道部分加上
```
ssh -i ~/.ssh/siyao.pem -L 127.0.0.1:3307:172.31.11.115:3306 ubuntu@44.200.72.244
```
整体的格式为
```
ssh -i <本地私钥路径> -L 起点地址:起点端口:终点地址:终点端口 用户名@跳板机地址
```
请注意，终点终点地址应该是内网的IP地址。命令的含义是进行SSH加密通信，并且建立SSH本地隧道，监听本地3307端口的流量。跳板机收到数据包后，会帮我们转发到数据库的3306端口中。我们实际上也可以把127.0.0.1去掉变成
```
ssh -i ~/.ssh/siyao.pem -L 3307:172.31.11.115:3306 ubuntu@44.200.72.244
```
这样就本地回环地址就不仅限于IPv4了。此外，我们可以使用`-f`来使隧道后台运行，用`-N`使得此窗口不执行远程命令，只做端口转发。
#### 使用隧道
当我们输入命令后，本地隧道就已经建好了。注意此时的SHELL是跳板机的。我们可以输入命令
```
mysql -h 127.0.0.1 -P 3307 -u testuser -p
```
接下来键入密码。这样就登录上Mysql数据库了。
#### 常见错误
如果连接失败，记得到跳板机的`/etc/ssh/sshd_config`文件中将`AllowTcpForwarding`改为`yes`即可。
### SSH远程隧道（反向隧道）
如果要想用公网服务器连接本地的服务器，这就需要远程隧道了。此时服务端和客户端互换，且本地服务器需要开启22号端口。**像这样从公网穿透到没有公网IP地址的电脑就是内网穿透。**
#### 建立隧道
我们可以从服务器发起请求吗？不行，因为家中路由器一般会屏蔽22号端口的请求。而且还有地址转换的问题。我们还是  应该从本地机入手
```
ssh -i ~/.ssh/siyao.pem -R 2200:localhost:22 ubuntu@44.200.72.244
```
这里的大体逻辑和本地隧道差不多，只是变为了两台机器的事情。
#### 使用隧道
```
ssh mac@localhost -p 2200
```
请注意这里一定要指定端口，不然默认走的是22号端口。
### 本地隧道和远程隧道的局限性
这两种隧道都是一对一的形式，每个隧道都需要本地机一个端口，如果连接多了很容易端口冲突。如果内网地址因为DHCP或者其他原因导致IP改变，又要全部重新配置连接。
### 动态隧道
首先在远程服务器`/etc/ssh/sshd_config`中将`GatewayPorts`设置为`yes`。
#### 配置隧道
```
ssh -i ~/.ssh/siyao.pem -D 1050 ubuntu@44.200.72.244
```
整体的格式为
```
ssh -i <本地私钥路径> -D 本地端口 用户名@服务器地址
```
这条命令的意思就是在本地机上监听1080端口，与此同时1080端口变成了SOCKS5代理服务器。任何支持SOCKS5代理的软件都能通过1080端口访问远程内网里的任意IP地址和对应端口。
#### 使用隧道
假设内网服务器中在8001端口部署了web服务器
```
curl --socks5-hostname localhost:1080 http://127.0.0.1:8001
```
这样就可以正常访问了。
