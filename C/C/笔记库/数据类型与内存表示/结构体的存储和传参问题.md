结构体的内存大小其实是比较特殊的，这就涉及到了结构体内存对齐。
## 结构体内存对齐
>1. 第一个成员在与结构体变量偏移量为0的地址处
2. 其他成员变量要对齐到某个数字（对齐数）的整数倍的地址处。对齐数=编译器默认的一个对齐数与该成员大小的较小值。 **VS的默认值为8 **
3. 结构体总大小为最大对齐数（每个成员变量都有一个对齐数）的整数倍
4. 如果嵌套了结构体的情况，嵌套的结构体对齐到自己的最大对齐数的整数倍处，结构体的整体大小就是所有最大对齐数（含嵌套结构体的对齐数）的整数倍
5. **特别注意，对齐数计算的时候看到是类型，与有几个这样的元素无关！例如`char a[10]`，该成员的对齐要求为1字节！**

上面是最基本的规则。接下来我们将进行详解。
### 例子
```C
#include <Stdio.h>
struct S1 {
	char c1;
	int num;
	char c2;
};
int main()
{
	int size = sizeof(struct S1);
	printf("%u", size);
}
```
```C
12
```
为什么会有这样的输出呢？VS的默认对齐数是8，而`char`是1，取较小的一个数那么对齐数为1。同样的，`int num`的对齐数是4，`char c2`的对齐数是1。接下来我们要开始存储：

### 为啥要存在内存对齐？
1. 平台（移植）原因：不是所有的硬件平台都能访问任意地址上的任意数据的；某些硬件平台只能在某些地址处取某些特 定类型的数据，否则抛出硬件异常。
2. 性能原因：数据结构(尤其是栈)应该尽可能地在自然边界上对齐。为了访问未对齐的内存，处理器需要作两次内存访问；而对齐的内存访问仅需要一次访问。假如电脑是32位的，电脑一次可以拿四个字节中的内容，如果对齐的话就可以更快的访问。
总而言之，这是一个用空间换时间的行为。
### 启发
所以我们在设计结构体的时候，让空间占用小的成员尽量集中在一起。
## pragma
pragma是一个编译器指令
1. `#pragma once`头文件只包含一次
2. `#pragma pack()`恢复到默认对齐数
3. `#pragma pack(<number>)`将对齐数修改为`number`。特别的，当`number`为1的时候，即为连续存储。
## 结构体的传参问题
类似于数组，结构体有两种传参方式。传址方式是较为推荐的。
### 传值
```C
#include <Stdio.h>
struct Score {
	int cn;
	int en;
};
void print1(struct Score s) {
	printf("%d %d", s.cn, s.en);
}
int main()
{
	struct Score s1 = {91, 78};
	print1(s1);
}
```
### 传址
```C
#include <Stdio.h>
struct Score {
	int cn;
	int en;
};
void print1(const struct Score* p) {
	printf("%d %d", p->cn, p->en);
}
int main()
{
	struct Score s1 = {91, 78};
	print1(&s1);
}
```
