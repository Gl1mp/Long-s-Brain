### 二维数组 
#### int a\[3]\[4] = {0};
```C
int a[3][4] = {0};
printf("%d\n",sizeof(a)); //1
printf("%d\n",sizeof(a[0][0])); //2
printf("%d\n",sizeof(a[0])); //3
printf("%d\n",sizeof(a[0]+1)); //4
printf("%d\n",sizeof(*(a[0]+1))); //5
printf("%d\n",sizeof(a+1)); //6
printf("%d\n",sizeof(*(a+1))); //7
printf("%d\n",sizeof(&a[0]+1)); //8
printf("%d\n",sizeof(*(&a[0]+1))); //9
printf("%d\n",sizeof(*a)); //10
printf("%d\n",sizeof(a[3])); //11
```

### 3
```C
int main()
{
	int a[4] = { 1, 2, 3, 4 };
	int *ptr1 = (int *)(&a + 1);
	int *ptr2 = (int *)((int)a + 1);
	printf( "%x,%x", ptr1[-1], *ptr2);
	return 0;
}
```
```C
4,2000000
```
这里的问题主要是在ptr1上。这是a被转化为int类型，在加法的时候只前进了一个字节。从此开始的连续四个字节被解释为一个整数。这里还有小端存储的问题
```C
01 00 00 00 02 00 00 00
   |         | 
   从此处开始  到此结束
```
然后因为有小端存储，翻转过来。以`%d`打印的时候，按照十六进制无符号整数打印。
### 4
```C
#include <stdio.h>
int main()
{
	int a[3][2] = { (0, 1), (2, 3), (4, 5) };
	int *p;
	p = a[0];
	printf( "%d", p[0]);
	return 0;
}
```
这个括号中其实是一个逗号表达式，值为最后一项。也就是说，显示初始化的有：
```C
{1, 3, 5}
```

|      | col0 | col1 |
| ---- | ---- | ---- |
| row0 | 1    | 3    |
| row1 | 5    | 0    |
| row2 | 0    | 0    |
后面的分析较为简单，`a[0]`为二维数组的第一个元素，也就是第一行这一个一维数组。那么p就是一维数组的第一个元素。
```C
1
```
