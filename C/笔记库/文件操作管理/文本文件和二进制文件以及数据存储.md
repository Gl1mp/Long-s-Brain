## 1. **基本概念：文本文件和二进制文件**

- **根据数据的组织形式分类**：
    - **文本文件**：数据以**ASCII码（或Unicode等字符编码）**形式存储在外存（磁盘）上。简单说，就是“人能读懂的字符形式”。比如，用记事本打开，能看到正常文字/数字。
        - 适合存储字符串、格式化数据（如日志、配置文件）。
        - 存储前需要**转换**：内存中的二进制数据转成ASCII字符。
    - **二进制文件**：数据直接以**内存中的二进制形式**输出到外存，没有转换。简单说，就是“机器读懂的原始二进制”。用记事本打开，会看到乱码（因为不是字符）。
        - 适合存储图像、音频、结构体、数值数组等原始数据。
        - 优点：存储效率高（体积小）、读写快；缺点：不可读性强。
- **为什么区分？**
    - 内存中，所有数据（包括字符、整数、浮点）都以**二进制**存储（计算机底层只有0和1）。
    - 输出到文件时：
        - **不转换** → 二进制文件（直接dump内存）。
        - **转换** → 文本文件（转成可见字符，如'1'的ASCII是49二进制）。

## 2. **数据在内存中的存储方式**

- **字符型数据**（char）：
    - 总是以**ASCII形式**存储在内存中（1字节/字符）。
    - 示例：字符 'A' 在内存中是二进制 01000001（ASCII码65）。
- **数值型数据**（int、float等）：
    - 在内存中以**二进制**存储（补码表示）。
    - 输出到文件时，可以选择：
        - **ASCII形式**：转成字符序列（e.g., 整数10000 → '1' '0' '0' '0' '0'，每个字符1字节，总5字节）。
        - **二进制形式**：直接存内存二进制（e.g., int 10000 在32位系统是4字节：十六进制 00 00 27 10）。
- **示例：整数10000的存储对比**
    - **内存中**：二进制形式，占4字节（int类型，通常32位）。
        - 二进制：00000000 00000000 00100111 00010000（小端序，视系统而定）。
    - **文本文件（ASCII）**：输出为"10000"，占5字节（'1'、'0'、'0'、'0'、'0'）。
        - 用fprintf/pf = fopen("test.txt", "w"); fprintf(pf, "%d", 10000); → 文件大小5字节。
    - **二进制文件**：直接写内存二进制，占4字节。
        - 用fwrite → 文件大小4字节。
    - **为什么体积不同？** ASCII是“可视化”存储，多占空间（每个数字转字符）；二进制是“紧凑”存储，节省空间。
