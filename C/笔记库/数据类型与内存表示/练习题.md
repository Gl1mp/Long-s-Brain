## 1
```C
#include <Stdio.h>
int main()
{
	//char  -128~127
	//unsigned char 0~255
	unsigned char a = 200;
	//00000000000000000000000011001000
	//11001000 -a 截断
	unsigned char b = 100;
	//00000000000000000000000001100100
	//01100100 - b 截断
	unsigned char c = 0;

	c = a + b;
	//整型提升
	//00000000000000000000000011001000
	//00000000000000000000000001100100
	//00000000000000000000000100101100
	//00101100-c 截断
	//00000000000000000000000000101100
	//
	printf("%d %d", a + b, c);
	//300 44
	return 0;
}
```
1. 我们发现无符号char中都能存储这些数字，也就是将他们转换为int下的32位，然后截断后八位即可
2. `a+b`的相加的时候会触发整型提升，因为都是无符号数，高位都用0填补
3. 得到结果`00000000000000000000000100101100`以%d解释打印出来第一个结果
4. 我们知道无符号整型`c`是由`a+b`得来的，类似于2，3：我们得到了`00000000000000000000000100101100`。接下来要存在无符号的char中，截断后八位。
5. 在打印的时候以`%d`输出，会发生默认参数提升，此时会有符号扩展。也就得到了`00000000000000000000000000101100`。注意，得到的这个是补码，如果是以有符号打印，如果最高位为补码，还需要取反加一转换为原码才能解读出正确的数值！
