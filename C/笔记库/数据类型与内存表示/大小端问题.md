## 字节序存储：大端与小端

存储分为两种：**大端字节序**（Big-Endian）和**小端字节序**（Little-Endian）。顾名思义，这样的存储是基于字节的。我们知道，两位16进制数对应的存储空间就是**1字节**。

假设我们要存储一个整型数（4字节），其十六进制表示为：

（高位）`19 34 48 91`（低位）

### 大端字节序（Big-Endian）
- **规则**：高位字节存放在低地址，低位字节存放在高地址。
- **存储顺序**：从低地址到高地址：`19 34 48 91`

### 小端字节序（Little-Endian）
- **规则**：低位字节存放在低地址，高位字节存放在高地址。
- **存储顺序**：从低地址到高地址：`91 48 34 19`

Visual Studio（在 x86/x64 架构下）使用的是小端字节序。

## 比较表格

| 字节序类型 | 规则描述     | 示例存储（低地址 → 高地址） |
| ----- | -------- | --------------- |
| 大端    | 高位字节在低地址 | 19 34 48 91     |
| 小端    | 低位字节在低地址 | 91 48 34 19     |
## 指针转换实现大小端判断
```
#include <Stdio.h>
int main() {
    int a = 1;
    char* p = (char*)&a;
    if (*p == 1) {
        printf("Little Endian\n");
    } else {
        printf("Big Endian\n");
    }
}
```

这个程序就可以实现了。原理很简单，就是原来的1为`00 00 00 01`，取一个字节，看里面放的是1或0就可以了。
